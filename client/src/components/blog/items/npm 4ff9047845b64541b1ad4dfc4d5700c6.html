npm how to
get the version of an installed packagenpm list <package-name>
list globally installed packagesnpm list -g npm ls --depth=0 -g
list local installed packagesnpm list npm ls --depth=0
create a package.json filenpm init
generate a package-lock filenpm install --package-lock-only
install a packagenpm i <package-name>
install version greater than or equal tonpm i @material-ui/pickers@">=4.0.0-alpha.8"
install a specific versionnpm install --save react@16.0.0
install in version rangenpm install sax@">=0.1.0 <0.2.0" npm install @myorg/privatepackage@"16 - 17"
install latest version regardless of your semver npm install --save react@latest
update an installed package to the latest version allowed by the semver in your package.jsonnpm up <package> npm up -g <package> <package> <package>So, if your package.json says "react": "^15.0.0" and you run npm up react your package.json will now say "react": "^15.6.2" (the latest current version of react 15)
update to the latest version, ignoring the semver in package.jsonnpm install --save react@latest
update to a specific version, ignoring the semver in package.jsonnpm install --save react@16.0.0
list all outdated local packagesnpm outdated
list all outdated global packagesnpm outdated -g --depth=0
list all package versions available for downloadnpm show <package> versions
list all the dependencies for a package versionnpm view react@17.0.0 dependencieshttps://docs.npmjs.com/cli/v8/commands/npm-view
list all packages where name, desc, author, date, latest version, or keywords include your search term npm search react
run test scripts from a menunpm t
upgrade npm on windowsnpm i -g npm@latest
upgrade npm on linuxnvm install-latest-npm
verify npm settings (can take a long time)npm doctor
view or change npm registry settingsnpm profile get [<key>] npm profile set <key> <value> Run "npm help profile" for more info
